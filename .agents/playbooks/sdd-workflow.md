# SDD Workflow

1. Constitution: keep `.specify/memory/constitution.md` aligned with project rules.
2. Specify: update `specs/001-yas-lab2-cd/spec.md` before adding new CD behavior.
3. Plan: update `specs/001-yas-lab2-cd/plan.md` when architecture or job behavior changes.
4. Tasks: update `specs/001-yas-lab2-cd/tasks.md` with concrete task status.
5. Implement: make scoped changes and verify after each increment.

Spec changes are part of the repo and should be committed with the implementation they govern.
